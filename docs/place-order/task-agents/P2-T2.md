Reserve API
Endpoint: POST /internal/v1/reservations
Checklist:
Input validation:

 idempotencyKey không rỗng
 items không rỗng, tối đa 50 items
 Mỗi item: productId > 0, quantity từ 1–1000

Idempotency:

 Query Reservations WHERE OrderIdempotencyKey = ? AND Status = PENDING
 Nếu tìm thấy → trả 201 với reservationId cũ (no-op)
 Nếu không → xử lý tiếp

Reserve logic (trong DB transaction):

 Validate tất cả productId tồn tại
 Với mỗi item: check Available >= quantity — nếu không đủ → throw OutOfStockException
 Available -= quantity, Reserved += quantity
 INSERT Reservation với ExpiresAt = UtcNow.AddMinutes(15)
 Commit

Optimistic lock:

 Catch DbUpdateConcurrencyException → reload entity, retry
 Tối đa 3 lần retry
 Sau 3 lần → throw ConcurrencyException → 503

Response:

 201: { reservationId, expiresAt, items[] }
 409: { type: "OUT_OF_STOCK", productId, requested, available }
 503: Khi concurrency retry exhausted
