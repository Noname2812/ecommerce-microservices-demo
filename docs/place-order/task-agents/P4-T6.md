Core Place Order Handler
Đây là task quan trọng nhất của toàn bộ project.
Checklist:
Orchestration flow:

 Step 1: Idempotency check (P1-T4 middleware)
 Step 2: Layer 1–3 validation (P4-T2)
 Step 3: Calculate pricing (P4-T3)
 Step 4: Reserve inventory (P4-T4)

 Catch OutOfStockException → 409 (không làm gì thêm)
 Catch InventoryUnavailableException → 503 (không làm gì thêm)


 Step 5: Claim coupon — chỉ khi có couponCode (P4-T5)

 Catch bất kỳ exception → ghi CompensationOutbox trước, rồi mới throw
 CompensationOutbox: IInventoryReleaseRequested { reservationId, reason: "COUPON_CLAIM_FAILED" }


 Step 6: Save order (transaction)

 BEGIN TRANSACTION
 INSERT Orders (Status=CONFIRMED, gắn reservationId + claimId)
 INSERT OutboxMessages (Type=IOrderConfirmed)
 COMMIT
 Nếu fail → ghi CompensationOutbox (separate connection, best-effort):

 IInventoryReleaseRequested { reservationId }
 ICouponReleaseRequested { claimId } (nếu có)




 Step 7: Cache idempotency response (Redis)
 Step 8: Trả 201

Logging requirements:

 Log mỗi step với correlationId, userId, latency
 Log ERROR khi compensation triggered với reason
 Log INFO khi order confirmed

Tests:

 TC-HP-001: Happy path không coupon
 TC-HP-002: Happy path có coupon
 TC-HP-003: Idempotency retry
 TC-INV-001: Inventory fail → 409, không save order
 TC-CPN-002: Coupon fail → 409, CompensationOutbox có IInventoryReleaseRequested
 TC-SAVE-001: DB fail → 500, CompensationOutbox có 2 events
 TC-SAVE-002: Retry sau crash → flow chạy lại bình thường
