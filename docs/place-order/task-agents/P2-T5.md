Checklist:

 Hangfire RecurringJob: chạy mỗi 2 phút
 Query: SELECT * FROM Reservations WHERE Status='PENDING' AND ExpiresAt < GETUTCDATE()
 Batch tối đa 200 records mỗi lần chạy
 Với mỗi expired reservation (trong transaction):

 Available += Quantity, Reserved -= Quantity
 Status = RELEASED, ReleasedAt = UtcNow


 Log: số lượng released mỗi lần, có correlationId = "ttl-job-{timestamp}"
 Xử lý đúng khi batch = 0 (không log gì hoặc log debug)
 Idempotency: dùng optimistic lock hoặc WHERE Status='PENDING' để tránh double-release nếu job chạy chồng lên nhau

Tests:

 Reservation hết hạn → được release, Available tăng
 Reservation chưa hết hạn → không bị release
