Validation Pipeline
Checklist:
Layer 1 — FluentValidation:

 PlaceOrderRequestValidator:

 UserId không empty
 Items không rỗng (1–20 items)
 Mỗi item: ProductId > 0, Quantity 1–100
 ShippingAddress: FullName, Phone (regex), Address, District, City bắt buộc
 CouponCode: optional, nếu có thì 1–64 ký tự alphanumeric+dash
 PricingSnapshot.CapturedAt không quá 30 phút trong quá khứ


 Global exception filter: trả 400 với danh sách lỗi có field name

Layer 2 — Rate Limit:

 Redis sliding window: rate:order:{userId} — max 5/phút
 Trả 429 với header Retry-After: 60
 (Sử dụng middleware từ P1-T4)

Layer 3 — Business Rules (Task.WhenAll):

 IProductValidator.ValidateAsync(items):

 Tất cả ProductId tồn tại trong catalog
 Tất cả IsActive = true


 IShippingValidator.ValidateAsync(address):

 City/District thuộc danh sách supported regions


 IPricingValidator.ValidateAsync(snapshot, items):

 Lấy giá hiện tại từ catalog
 So sánh với snapshot: tolerance ±1%
 Nếu chênh → 422 PRICE_MISMATCH với currentPrice



Tests:

 Mỗi validator rule có ít nhất 1 test pass + 1 test fail
 Task.WhenAll: khi 1 rule fail → trả đúng error, không chờ rule khác
